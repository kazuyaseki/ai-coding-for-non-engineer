## 第3章：自分だけのAIパートナーを作ろう

### Gemini Chatアプリの作成（メインプロンプト）

```markdown
Vite + Vanilla JSを使って、以下の要件を満たす「Gemini Chat」アプリケーションを作成してください。

## デザインコンセプト
- **テーマ**: スタイリッシュで退廃的なモノクロデザイン（SFチックなシステム画面風）
- **配色**:
    - 背景（外側）: 光沢のある白 (#f0f2f5) とラジアルグラデーション
    - アプリ背景: 透過性のあるダークグレー (#2b2b2b)
    - テキスト/アクセント: ベージュ (#dad4bb)
- **特徴**:
    - 背景に幾何学的なSVGアニメーション（回転する円とグリッド）を配置
    - ロゴは "GEMINI // SYSTEM"（Orbitronフォントを使用）
    - ロボットの絵文字ではなく、幾何学的なSVGアイコンを使用
    - スキャンラインや角ばったボーダーによる「システム感」を演出

## 技術スタックと機能
- **フレームワーク**: Vite (Vanilla JS)
- **API**: # Google Gen AI SDK for TypeScript and JavaScript (`@google/genai`)
- **モデル**: `gemini-3-flash-preview`
- **機能**:
    - テキストチャット（ユーザー vs AI）
    - 日本語入力のIME確定（Enter）で送信されないように制御
    - ローディング表示（"演算中..."）

## 実装のガイドライン（重要）

1. **パッケージ**: `@google/genai` を使用する（`@google/generative-ai`は使用しない）。
2. **初期化**: `const ai = new GoogleGenAI({ apiKey: ... })` の形式で行う。
3. **チャット機能**: `ai.models.generateContent` で履歴を自作するのではなく、**`const chatSession = ai.chats.create({ model: 'gemini-3-flash-preview' })` を使用してステートフルなチャットセッションを作成する**こと。
    - ドキュメントの #Chat 部分を参照してください
	- この時 config は何も書かないこと

## 作業の注意点
- npm install は後ほど自分でやります
- パッケージのバージョン指定は vite, @google/genai 共に `"latest"` または最新の安定版を使用すること
- .env.example ファイルを作って、その中に `VITE_GEMINI_API_KEY=YOUR_API_KEY` と書いてください
- .gitignore ファイルを作って、そこに .env と書いてください。

この要件に基づいて、プロジェクト全体を作成してください。

※作業を始める前に**必ず**以下のURL先を読んで、最新のGoogle Gen AI SDK仕様に厳密に従ってください：
https://github.com/googleapis/js-genai/blob/main/codegen_instructions.md
```

## 第13章：AIエージェント開発

### Tool Calling機能の追加

```
以下の要件に従って、main.jsに「システムの色を変更するTool Calling機能」を追加してください。

## 機能要件
- ユーザーが「色を赤にして」のように指示すると、アプリのアクセントカラー（`--color-accent`, `--color-border`）を変更するツールを実行する。
- ツール名: `change_theme_color`
- 引数: `color` (文字列, hexコードまたは色名)

## 実装の手順（@google/genai SDK v1仕様）

1. **ツールの定義**:
   - `Type` を `@google/genai` からインポートする。
   - `change_theme_color` ツールを定義し、`ai.chats.create` の `config.tools` に設定する。

2. **実行ロジック**:
   - クライアントサイドで実際にCSS変数を変更する関数 changeThemeColor(color) を実装する。

3. **Function Callingのループ処理**:
   - `chatSession.sendMessage` のレスポンスに `functionCalls` が含まれている場合、ツールを実行するループ処理を追加する。
   - **重要**: ツールの実行結果をモデルに返す際は、SDKの仕様に従い `functionResponse` を **配列 (`message` プロパティ)** にラップして送信してください。

   ```javascript
   // 正しい書き方の例
   const content = [{
       functionResponse: {
           name: "change_theme_color",
           response: { result: toolResult }
       }
   }];
   
   result = await chatSession.sendMessage({
       message: content
   });
```
